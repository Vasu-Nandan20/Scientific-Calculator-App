# Contributing to Numix

First off, thank you for considering contributing to Numix! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
- Use a clear and descriptive title for the issue.
- Describe the exact steps which reproduce the problem.
- Explain which behavior you expected to see instead and why.

### Suggesting Enhancements
- Explain why this enhancement would be useful to most Numix users.
- Provide step-by-step descriptions of the suggested enhancement.

### Pull Requests
- Fill in the required template.
- Ensure the code follows the existing style of the project.
- Update the documentation if you're making changes to the UI or adding new features.

## Style Guide
- Follow [Kotlin coding conventions](https://kotlinlang.org/docs/coding-conventions.html).
- Use descriptive variable and function names.
- Keep functions small and focused on a single task.

Thank you for your contribution!
